addRouteRequirement

(
  • name
  • fn
)

Defined in src/Router/Router.js:467

Set a function as the request handler for a service. If a route is registered to the specified service, then this function will be called with an instance of the Request and Response objects before calling the controller. Firefly will prepend the

Parameters:

  • name String

    Name of the rule. An _ (underscore) is prepended to the name

  • fn Function

    listener function to be used as the request handler

Example:

   addRouteRequirement('useraccess', function(req, res, rule, fn) {
       fn(true) //passes test
       fn(false, true) //failed test. hault routing. the service will take care of showing error touser
       fn(false, false) // failed test. continue routing
   });


   //In routes file useraccess will be declared as:
   _useraccess: 'admin'
   //thus the value of rule in the service handler will be 'admin'
